## 2. 主要遷移フロー

### 2.1 オンボーディング
```
[LP/外部導線]
    ↓
[AUTH-001 ログイン・アカウント作成]
    ├─ 新規登録 → [AUTH-002 プロフィール設定] → [COMMON-001 ダッシュボード]
    ├─ ログイン成功 → [COMMON-001 ダッシュボード]
    └─ ゲスト開始 → [UC003-001 ポータルトップ] (機能制限あり)
```

### 2.2 デートプラン生成〜共同編集
```
[UC003-001 ポータルトップ]
    ↓ 「AIデートを作成」
[UC001-001 デートプラン作成]
    ↓ AI生成開始
[UC001-002 AI生成中]
    ↓ 完了
[UC001-003 プラン提案]
    ├─ プラン確定 → [UC001-004 プラン詳細]
    └─ 再生成/フィードバック → [UC001-001]
[UC001-004 プラン詳細]
    ├─ カスタマイズ → [UC001-005 カスタマイズビュー]
    └─ 共同編集 → [UC002-001 共同編集ボード]
```

### 2.3 Conflict Mediation（喧嘩仲裁）
```
[COMMON-001]
    ↓ 「関係サポート」
[UC004-001 仲裁依頼]
    ↓
[UC004-002 状況分析]
    ↓
[UC004-003 仲裁提案]
    ├─ 継続ケア → [UC004-004 関係改善プラン]
    └─ レポート → [UC004-005 振り返りレポート]
```

### 2.4 マネタイズ導線
```
[UC003-001 / COMMON-001]
    ↓ 「プレミアム機能」
[UC007-001 課金管理]
    ├─ プラン変更 → [UC007-002 機能解放]
    ├─ 課金通知設定 → [UC007-003 課金変更通知]
    ├─ 売上分析 → [UC007-004 課金指標分析]
    └─ サブスク管理 → [UC007-005 サブスクリプション管理]
```

### 2.5 Date Canvas 連携
```
[UC001-004 / UC002-001]
    ↓ 「思い出を残す」
[UC005-001 Date Canvas]
    ├─ メモ編集 → 保存
    └─ 共有 → 共有リンク発行
```

## 3. ナビゲーション仕様メモ
- 全画面共通のグローバルナビ: ログイン、ポータル、ダッシュボード、ヘルプ（`../common/COMMON-003.html`）。
- フッターメニュー: AIデートプラン、共同編集、ポータル、設定、ヘルプ。
- UC005-002 / UC005-005、COMMON-005 はバックログ扱い。画面一覧の表ではステータスを明記し、遷移図からは除外しています。
- 画面遷移は `Prototype/js/navigation.js` の `navigateTo(url)` を標準化。SPA 化時も同じルーティングIDを利用予定。

## 4. 実装フェーズで必要な差分
1. `Prototype/screens/common/COMMON-002.html` 以降の共通画面は UI & 文言の最終確定が未着手。実装時は当資料のステータスを参照しつつ詳細仕様書を補完してください。
2. バックログ扱いの画面はナビゲーション上からリンクを出さない運用とし、将来リリース時には本資料を更新します。
3. 本ドキュメントは `screensdocs` の画面仕様書と連動させて維持してください。
